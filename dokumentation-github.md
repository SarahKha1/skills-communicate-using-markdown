# Dokumentation – Introduction to GitHub und Communicate using Markdown

## Introduction to GitHub

In der ersten Übung habe ich gelernt, wie GitHub funktioniert.
Ich habe gesehen, wie man ein Repository benutzt, Branches erstellt, Commits macht und einen Pull Request öffnet und merged.

**Branch erstellen**
Ich bin im Repository auf den Code-Tab gegangen und habe einen neuen Branch angelegt.
Der Name war `my-first-branch`.
Damit konnte ich Änderungen machen, ohne dass die Hauptversion `main` verändert wird.
So kann man sicher arbeiten.

**Datei erstellen und committen**
Dann habe ich eine neue Datei erstellt und dort etwas geschrieben.
Zum Beispiel:

```
# Mein erstes GitHub-Dokument
Hier teste ich, wie Commits funktionieren.
```

Danach habe ich einen Commit gemacht.
Ein Commit speichert den aktuellen Stand und bekommt eine kurze Beschreibung wie
„Add index.md file“.

**Pull Request öffnen**
Als Nächstes habe ich einen Pull Request geöffnet.
Damit schlägt man vor, seine Änderungen in den Hauptzweig einzufügen.
Im Pull Request sieht man genau, was sich geändert hat.

**Pull Request mergen**
Zum Schluss wurde der Pull Request gemerged.
Damit wurden meine Änderungen aus dem Branch `my-first-branch` mit dem Hauptzweig `main` zusammengeführt.
Danach war alles abgeschlossen und mein Text war offiziell gespeichert.

## Communicate using Markdown

In der zweiten Übung habe ich Markdown gelernt.
Markdown ist eine einfache Sprache, mit der man Texte auf GitHub formatiert.
Man braucht keine zusätzlichen Programme, sondern nur bestimmte Zeichen.

**Überschriften hinzufügen**
Ich habe gelernt, wie man Überschriften schreibt:

```
# Überschrift 1
## Überschrift 2
### Überschrift 3
```

Damit kann man den Text gut strukturieren.

**Bild einfügen**
Ich habe ein Beispielbild hinzugefügt:

```
![Yaktocat](https://octodex.github.com/images/yaktocat.png)
```

Das zeigt ein Bild der Figur Yaktocat.
Der Text in den eckigen Klammern ist der Alt-Text, der angezeigt wird, wenn das Bild nicht geladen werden kann.
So sieht es aus:

![Yaktocat](https://octodex.github.com/images/yaktocat.png)

**Codebeispiel einfügen**
Ich habe gelernt, wie man Code in Markdown richtig zeigt.

````
```javascript
var myVar = "Hello, world!";
console.log(myVar);
````

```
Damit kann man Code übersichtlich darstellen, und GitHub zeigt ihn an.

**Aufgabenliste erstellen**  
Dann habe ich eine Aufgabenliste gemacht. 
Das geht mit eckigen Klammern:
```

* [ ] Branch erstellen
* [x] Bild hinzufügen
* [x] Codebeispiel einfügen
* [ ] Pull Request mergen

```
So sieht die Liste in Markdown aus:

- [ ] Branch erstellen  
- [x] Bild hinzufügen  
- [x] Codebeispiel einfügen  
- [ ] Pull Request mergen  

Man kann dort abhaken, was schon erledigt ist.
```

**Pull Request mergen**  
Zum Schluss habe ich wieder alles gespeichert, committed und gemerged.
